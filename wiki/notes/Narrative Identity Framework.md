# Narrative Identity Framework

A proposed (not confirmed-adopted — see [[Narrative Identity]]'s status note) design methodology for giving every FEARS class a distinct non-combat story footprint, on top of its existing mechanical and thematic identity.

## Three-layer model

1. **Mechanical Identity** — what the class does in combat; its loop. Already strong in FEARS per the source.
2. **Thematic Identity** — the fantasy promise and emotional tone. Also already strong.
3. **Narrative Identity** — the recurring story pressure, relationships, and consequences the class generates. The layer this framework targets.

Test: "A class has narrative identity when the system helps create class-specific story problems, not just class-specific combat choices."

## The six parts

1. **Narrative Promise** — what kind of story/arc the class naturally invites. Test: remove combat entirely — does the class still imply a distinct story?
2. **Narrative Pressure** — the ongoing tension that follows the class even when not chosen. The most important part per the source.
3. **Narrative Currency** — what the class gains, spends, risks, or accumulates in story terms (e.g. [[Spirit Debt]], [[Territory Bond]]). Distinguishes one class's non-combat arc from another's.
4. **World Reaction** — how people, factions, places, and the setting respond to the class specifically, before any dice are rolled.
5. **Story Triggers** — recurring scene types the class should pull into a campaign (3-5 suggested per class).
6. **Narrative Reward** — what in-fiction behavior the system should reinforce as "playing the class right," beyond XP.

## Implementation levels (lightest to heaviest)

1. **Flavour Guidance Only** — prose in class intros/sidebars, no new systems.
2. **Soft Narrative Hooks** — suggested triggers/rewards, GM-adjudicated.
3. **Light Mechanics** — one class-specific currency or pressure track (even a 0-3 track is enough).
4. **Full Narrative Integration** — classes directly interact with [[Sanity]], [[Focus]], [[Corruption]], faction standing, downtime, exploration, and recovery.

## Risks named in the source

- **Overdetermination** — mandating pressures every time ("every Witch must be hunted") becomes restrictive rather than evocative. Fix: common and meaningful, not mandatory.
- **Mechanical bloat** — a full subsystem per class overloads the game. Fix: one clear pressure or currency per class, not five.

## Recommendation given for FEARS

Every class gets 1 Promise + 1 Pressure + 1 World Reaction note + 3-5 Story Triggers. Only 8 classes get a full currency track: Witch, Sangromancer, Reaper, Investigator, Warden, Shaman, Inventor, and maybe Warlock — chosen because they "already imply consequence-heavy identities."
